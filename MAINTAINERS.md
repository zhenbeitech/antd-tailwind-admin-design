# 维护者指南

本文档为 antd-tailwind-admin-design 的维护者提供指导。

## 维护职责

### 版本管理
- **主版本号(1.x.x)**: 重大变更或不兼容更新
- **次版本号(x.1.x)**: 新功能添加，向后兼容
- **修订号(x.x.0)**: 错误修复，向后兼容

### 发布流程

1. **更新 CHANGELOG.md**
   - 记录所有重要更改
   - 按版本号排序
   - 描述具体变更内容

2. **更新 package.json**
   - 更新 version 字段
   - 更新日期和相关信息

3. **创建 Git 标签**
   ```bash
   git tag -a v1.0.0 -m "版本 1.0.0"
   git push origin v1.0.0
   ```

4. **发布到 npm**
   ```bash
   npm publish
   ```

## 质量保证

### 代码审查 checklist
- [ ] 所有新功能都有相应的文档
- [ ] 示例代码可以正常运行
- [ ] 遵循 DESIGN.md 中的设计规范
- [ ] 不引入硬编码的样式值
- [ ] 正确使用 AntD Token
- [ ] 考虑了暗色模式适配
- [ ] 没有破坏性变更（除非是主版本）

### 测试要求
- [ ] 示例代码通过 TypeScript 编译
- [ ] 组件示例符合实际使用场景
- [ ] 页面模板包含完整的生命周期
- [ ] 样式在不同主题下正常显示

## 文档维护

### DESIGN.md 更新规则
- 9节结构保持不变
- 新增内容必须放在相应章节
- 删除过时的内容时保留历史记录
- 更新示例代码时确保准确性

### README.md 更新规则
- 安装方式保持最新
- AI 使用指引保持清晰
- 文件索引保持同步

## 社区互动

### Issue 处理
- **Bug 报告**: 24小时内响应，7天内解决
- **功能请求**: 评估可行性并给出反馈
- **文档问题**: 优先处理准确性问题

### PR 审核
- 确保符合代码规范
- 验证功能完整性
- 检查文档更新
- 确保向后兼容

## 技术债务管理

### 优先级
1. **P0**: 严重 bug、安全漏洞
2. **P1**: 功能缺失、不兼容变更
3. **P2**: 性能优化、代码重构
4. **P3**: 文档完善、示例补充

### 重构原则
- 保持向后兼容
- 更新相应的文档和示例
- 添加必要的测试
- 记录在 CHANGELOG.md 中

## 性能监控

### 关键指标
- 包大小增长
- 构建时间变化
- 文档加载速度
- 示例运行性能

### 优化措施
- 定期检查 bundle 大小
- 使用 Tree Shaking
- 优化图片和静态资源
- 缓存策略优化

## 安全考虑

### 依赖安全
- 定期检查依赖漏洞
- 使用 `npm audit` 进行扫描
- 及时更新有安全风险的包

### 输入验证
- 所有示例数据需要验证
- 避免潜在的 XSS 风险
- 使用 AntD 的安全特性
